看到 <font style="color:#DF2A3F;">原地操作 </font>应该就要想到栈。

用`low` 表示入栈的元素（即非零元素）

先遍历一次数组，把所有非零元素入栈。

然后统计`low` 的大小，把剩余的位置，填入0

```python
class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        low=0 #用来记录下标，把不为零的元素放在这里
        for i in range(len(nums)):
            if nums[i]!=0: # 找到一个不为零的元素
                nums[low]=nums[i] #把不为零的元素放过去
                low+=1
        for i in range(low,len(nums)):
            nums[i]=0
```

## [11. 盛最多水的容器](https://leetcode.cn/problems/container-with-most-water/) 26/3/3
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1771486523885-8f300cfc-6f44-488f-aab5-08d881d0d5ca.png)

不难发现，容器的大小由 长度和高度 两者决定。

在长度一定的时候，容量由最矮的一边来决定（木桶效应）

所以可以用双指针来枚举长度的两端 i,  j，如果`height[i]<=height[j]` 也就说明，是 i 端拉后腿了，尝试往后遍历，试图找一个更大的`height[i]` ；同理，若`height[i]>height[j]` 说明 j 拉后腿了，要往前遍历，试图找一个更大的`height[j]` 

```python
class Solution:
    def maxArea(self, height: List[int]) -> int:
        ans=0
        i,j=0,len(height)-1
        while i!=j:
            ans = max(ans,min(height[i],height[j]) * (j-i))
            if height[i]<=height[j]:
                i+=1
            else:
                j-=1
        return ans
```

## [15. 三数之和](https://leetcode.cn/problems/3sum/) 26/3/3
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1771470501740-9e256fa3-cdab-490a-8b04-094a1d011f9f.png)

思路和上一题有点像。

如果同时枚举三个数，三重循环复杂度太高。所以可以考虑用双指针来降低复杂度。

用for循环遍历一层 i ，作为枚举第一个数，然后用两个指针`left` `right` 表示另外两个数。

设置`left=i+1`, `right=n-1`

若三数之和大于0，`right-=1`; 反之 `left+=1`

找到相等的，就可以判断是否重复，然后 append，代码如下。

```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        if len(nums)<3 or nums[0]>0 or nums[len(nums)-1]<0:
            return []
        if len(nums)==3:
            if nums[0]+nums[1]+nums[2]==0:
                return [[nums[0],nums[1],nums[2]]]
            else:
                return []

        ans=[]
        for i in range(len(nums)-2):
            if nums[i]>0:
                break
            k=i+1
            j=len(nums)-1
            while k<j:
                if nums[k]+nums[j]<-nums[i]:
                    k+=1
                if nums[k]+nums[j]>-nums[i]:
                    j-=1
                if nums[k]+nums[j]+nums[i]==0 and i!=k and k!=j:
                    if [nums[i],nums[k],nums[j]] not in ans: 
                        ans.append([nums[i],nums[k],nums[j]])
                    k+=1
                    j-=1

        return ans
```

但是，用`if not in `来判断，是复杂度很高的（它每次判断都要遍历一次！）

能不能用剪枝的思路，来进行去重呢？

当然可以！

想象一下，如果给出一个数组 -4,-1,-1,0,1,1,2,3,5

+ 当 i 指向下标1和下标2（假设下标从0开始）时，数字都是-1，**那么i指向下标2时，所有三数之和为0的情况，肯定包含在下标1中**。所以for循环的时候，就可以把下标2跳过！ 也就是去重！
+ **同理，对于left，right 也一样**。
    - 当找到了三数之和为0的时候，要继续对left，right去重。

代码如下

```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        n=len(nums)
        ans=[]
        
        for i in range(n-2):
            if nums[i]>0: # 如果第一个都大于零，三数之和肯定大于0
                break
            if i!=0 and nums[i]==nums[i-1]: # 用来去重。
                continue
            left,right=i+1,n-1

            while left<right:
                s=nums[i]+nums[left]+nums[right]
                if s<0:
                    left+=1
                if s>0:
                    right-=1
                if s==0:
                    while left<right and nums[left]==nums[left+1]:
                        left+=1
                    while left<right and nums[right]==nums[right-1]:
                        right-=1
                    ans.append([nums[i],nums[left],nums[right]])
                    left+=1
                    right-=1

        return ans

```

思路：

1. 首先确定`dp[i]`的含义：`dp[i]`表示爬到第i层，有`dp[i]`种方式
2. 然后确定递推公式：  
若在 i-1 阶，到 i 需要再爬一阶；若在 i-2 阶，到 i 需要再爬两阶。  
所以，`dp[i] = dp[i-1] + dp[i-2]`
3. 初始化：  
`dp[1]=1` , `dp[2]=2`

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n<=2:
            return n
        dp = [0]*(n+1) #注意学习这种初始化的方式
        dp[1]=1
        dp[2]=2
        for i in range(3,n+1):
            dp[i]=dp[i-1]+dp[i-2]
        return dp[n]
```

## [118. 杨辉三角](https://leetcode.cn/problems/pascals-triangle/) 26/3/3
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1772505768996-37cc4be7-2a61-4df8-a440-2acb4d8157fd.png)

思路：

1. 首先确定`dp[i][j]`含义：  
`dp[i][j]` 表示对应位置的值
2. 递推公式：  
不难发现，`dp[i][j] = dp[i-1][j]+dp[i-1][j-1]`
3. 初始化：
    1. 每行的第一个和最后一个元素，要初始化为1。方便起见，我们直接把每行元素都初始化为1即可。因为由递推公式可知，每个中间的元素，都会重新计算并赋值，所以不需要担心初始化为1（而不是0）的影响。
    2. **注意要用列表推导式初始化**

```python
class Solution:
    def generate(self, numRows: int) -> List[List[int]]:
        '''
        注意这里要用列表推导式。
        如果直接: c=[[1]*n] *m,这样子是浅拷贝。修改其中一行的值后，会影响所有行。
        '''
        c = [[1]*(i+1) for i in range(numRows)] 

        for i in range(2,numRows):
            for j in range(1,i):
                c[i][j]=c[i-1][j-1]+c[i-1][j]

        return c
```

## [198. 打家劫舍](https://leetcode.cn/problems/house-robber/) 26/3/4

![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1772606438790-cf70c533-50e6-4eba-9e9c-16f80dbbbd55.png)

思路：

两个相邻的房间不能同时进入。所以定义`dp[i]`表示到第 i 个房间的时候的最大收入。

那么，`dp[i]`只有两种可能性：

+ 选择第 i 个，也就是没选择上一个 `dp[i]=dp[i-2]+nums[i]`
+ 选择了上一个 `dp[i]=dp[i-1]`

```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        dp=[0]*len(nums)
        dp[0]=nums[0]
        n=len(nums)
        if n==1:
            return dp[0]
        dp[1]=max(nums[0],nums[1])
        for i in range(2,len(nums)):
            dp[i] = max(dp[i-1],dp[i-2]+nums[i])
        return dp[n-1]
```

## [279. 完全平方数](https://leetcode.cn/problems/perfect-squares/) 26/3/4
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1772609324447-f59c40d9-f9f0-4756-b469-5615f5fe3ec9.png)

思路：

其实这道题是一个完全背包的问题。

翻译一下就是，**对一个容量为 n 的背包，有物品（平方数）1,4,9...。每种物品可以选择无限次，问最少选择几个物品，能够把背包恰好装满**。

然后就是几个小细节需要注意：

+ dp初始化的时候，要初始化为最大，然后注意加上float，才是整型
+ 遍历物品的时候，直接遍历平方根，然后再做平方！（本来我想用数组记录下来所有的平方数，太麻烦了）

```python

class Solution:
    def numSquares(self, n: int) -> int:
        dp=[0]+[float('inf')]*(n+1)

        for i in range(1,int(n**0.5)+1): #遍历物品，1,4,9...的平方根即1,2,3,...
            square = i*i #计算平方
            for j in range(square,n+1): #把物品填入背包
                dp[j]=min(dp[j],dp[j-square]+1) #若该物品填入了背包，就加一个，记录最小值

        return dp[n]
```

```python
class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        dp=[0]+[float('inf')]*(amount)
        for i in range(len(coins)):
            for j in range(coins[i],amount+1):
                dp[j]=min(dp[j],dp[j-coins[i]]+1)

```

## [139. 单词拆分](https://leetcode.cn/problems/word-break/) 26/3/5
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1772681842742-bc7b314f-e706-4f75-a8b9-5fd1bc411454.png)

思路：

**其实这道题也可以转化为完全背包问题。**

背包容量就是字符串长度；物品就是字符串列表。题目翻译过来就是，给定容量为`len(s)`的背包，物品为`wordDict`，物品可以无限次使用，请问物品能不能恰好装满背包？

但是有个坑！

对于**常见的背包问题，它是不需要考虑物品放进去的顺序的**！是组合问题！也就是说，先放物品1和先放物品2其实没有影响。所有我们完全可以外层循环枚举物品，先把物品1全部放进去，然后放物品2......

但是这道题明显不行。  
比如输入为：s = "applepenapple", wordDict = ["apple", "pen"]  
这个时候，如果先枚举物品，就是先把'apple'全部放进去，不难发现第一个匹配到了`s[0:5]`，但是下一个`s[8:13]`，虽然也是'apple'，却没办法匹配，因为在dp数组中，前面还有个'pen'没有匹配到！

**所以这道题是排列的问题！需要考虑顺序。**

**要在外层枚举背包容量！然后内层枚举物品！**

```python
class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        n=len(s)
        # dp[i] 表示字符串 s 的前 i 个字符能否被字典中的单词拼出
        dp=[False]*(n+1)
        dp[0]=True

        # 外层循环：遍历字符串的所有前缀长度（从左到右依次计算）
        for i in range(1,n+1):
            for word in wordDict:
                wlen = len(word)
                # 能够成功拼出的条件：
                # 1. 当前截取的长度必须够这个单词的长度 (i >= word_len)
                # 2. 刨去这个单词后，前面的前缀必须也是合法的 (dp[i - word_len] == True)
                # 3. 截取出来的这一段正好等于这个单词 (s[i - word_len:i] == word)
                if i>=wlen and dp[i-wlen] and s[i-wlen:i]==word:
                    dp[i]=True
        return dp[n]
```

## [<font style="color:rgb(0, 122, 255);">300. 最长递增子序列</font>](https://leetcode.cn/problems/longest-increasing-subsequence/) 26/3/5

![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1772685197913-d367ea7f-bbfd-466b-a4f6-21dd40da83f5.png)

整体思路： **站在当前位置，****往前找****比自己小的**。看看前面谁的队伍最长，我就排在谁后面。  

定义`dp[i]`表示以 i 为结尾的，前 i 个数的最长子序列的长度

所以，外层循环就枚举 i，内层就枚举 i 之前的。

代码如下：

```python
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        n=len(nums)
        dp=[1]*n

        for i in range(n):#以i为结尾
            for j in range(i):#找i之前的
                if nums[j]<nums[i]:
                    dp[i] = max(dp[i],dp[j]+1)
        return max(dp)
```

为什么内层循环只需要判断一次，就能确定dp[i]是前i个最大的？

因为dp数组是从前往后更新的，比方说i=5，j从0到5开始遍历，此时的dp[0]-dp[4]都已经是确定好了最大长度。所以只需要一个个跟nums[5]比较就好。

## [152. 乘积最大子数组](https://leetcode.cn/problems/maximum-product-subarray/) 26/3/6
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1772761034363-9584021e-a427-4732-90c0-0f825447b12e.png)

思路：

像子序列一样，我们**定义**`**dp[i]**`**数组为，以 i 为结尾的前 i 个数组的最大乘积**。

但跟子序列问题不同的是，子数组要求是连续的，所以不能用两重循环枚举。

既然必须是连续的。**那么**`**dp[i]**`** 要么包括第 i-1 个，要么就不包括（也就是从第 i 个重新开始计算）**

所以`dp[i]=max(dp[i],dp[i-1]*nums[i]`? 对吗？

不对！ 因为这是乘积，符号是有影响的！

那应该怎么办？要数一数有多少个负号吗？

其实不用，因为

+ **若**`**nums[i]**`**是正数，那么它要得到最大值，就要乘以一个最大的正数**
+ **若**`**nums[i]**`**是负数，那么它要得到最大值，就要乘以最小的负数**

**无非上面两种情况，我们把两种情况都维护下来就好。**

代码如下：

`maxdp[i]=max(nums[i],maxdp[i-1]*nums[i],mindp[i-1]*nums[i])`

`mindp[i]=min(nums[i],maxdp[i-1]*nums[i],mindp[i-1]*nums[i])`

由于只用到了 i-1，所以可以压缩，用 cur 记录上次的最值。

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        n=len(nums)
        ans=mindp=maxdp=nums[0] #ans是记录全局的最大值，maxdp是以i结尾的最大值

        for i in range(1,n):
            cur_maxdp,cur_mindp=maxdp,mindp
            maxdp=max(nums[i],cur_maxdp*nums[i],cur_mindp*nums[i])
            mindp=min(nums[i],cur_maxdp*nums[i],cur_mindp*nums[i])
            ans=max(ans,maxdp)
        # print(maxdp,ans)
        return ans
```

## [<font style="color:rgb(0, 122, 255);">416. 分割等和子集</font>](https://leetcode.cn/problems/partition-equal-subset-sum/) 26/3/6
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1772768304330-2e29d079-e975-46d7-b783-731d6e9cb488.png)

这道题思路还是挺巧妙的，我没想到hhh

其实可以转化为01背包问题！分割成两个相等的子集，也就是能不能在数组中选出来不重复的数，使得总和为`sum/2`!

那么代码就好说了

```python
class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        total_sum=sum(nums)
        n=len(nums)
        if total_sum%2!=0:
            return False
        target=total_sum//2
        if max(nums)>target:
            return False
        dp=[True]+[False]*target
        
        for i in range(n):
            for j in range(target,nums[i]-1,-1): #注意一维的01背包要倒着枚举
                dp[j]=dp[j] or dp[j-nums[i]]

        return dp[target]
```

有几点需要注意：

+ 如果总和是奇数，那么肯定不可能分成两个相等的子集
+ 如果有一个元素值大于`sum/2`，那么也肯定不行
+ 一个除号`/`是默认浮点数，两个`//`是整型

还有**01背包倒着枚举**的问题，这里详细说一下。

因为01背包每个元素只能取一次，如果正着枚举，就可能会重复，下面举个例子：

假如`nums[i]=2`,那么初始时`dp=[1,0,0,0,0]`  
若正着来，则当`j=2`时，`dp[j]=dp[j] or dp[j-nums[i]]`执行，使`dp[2]=dp[0]=1`。这个时候`nums[i]`已经被用了，不能再用了；  
但是当`j=4`时，`dp[4]=dp[2]=1`，相当于又用了`nums[i]`。

记忆的话，就直接和完全背包反着来就行。**因为完全背包是无限次的，要正序遍历；那么01背包只有一个，所以只能反着来。**

## [3. 无重复字符的最长子串](https://leetcode.cn/problems/longest-substring-without-repeating-characters/) 26/3/9
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1773022874376-eb05ddc4-7f85-4298-a740-0312b2fa27e4.png)

思路：

既然是滑动窗口，要怎么来模拟滑动呢？就是两个指针嘛，left, right 分别指向父串中子串的起始位置。  
判断`s[right]`是不是能够加入子串，并通过`left`和`right`调整子串大小。

对于本题，思路就很明确了：

当`s[right]`在子串出现过，就`left++`（即右移动）也就是更新子串，直到`s[right]`不在子串中出现。

这时，直接把`s[right]`也放进子串就好。

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        left,right=0,0
        n=len(s)
        ans = 0
        c=set() #这里用set集合来记录子串，查找起来更快,O(1)

        for right in range(n):
            while s[right] in c:
                c.remove(s[left]) # 移除left
                left+=1
            c.add(s[right]) #加入right
            ans = max(ans,right-left+1)
        return ans

```

## [438. 找到字符串中所有字母异位词](https://leetcode.cn/problems/find-all-anagrams-in-a-string/) 26/3/9

思路：

**就是像上一题一样，用**`**left,right**`**枚举子串的起始位置。然后对于每个子串，都判断一下是不是异位词**

但是怎么来判断？是有说法的。

因为异位词是不考虑顺序的，也就是只要字母完全一致就好。

那么就想到了排序（显然不是最优解...）只要把枚举到的子串和目标串p都排好序了，那么不就好比较了吗

代码：

```python
class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        m,n=len(s),len(p)
        ans = []
        if m<n:
            return []
        p=sorted(p) #注意字符串没有.sort()

        left,right=0,n-1
        for right in range(n-1,m):
            c = s[left:right+1]
            if sorted(c) == p:
                ans.append(left)
            left+=1
        return ans
```

但是这样时间复杂度太高了。

有没有更高效的方法？有！

因为异位词不需要考虑顺序，也就意味着，我们**只需要统计子串中字母出现的次数就行了**！

只要两个串，26个字母，每个字母出现的顺序都一样，那不就是异位词嘛！

所以**只需要维护两个列表，分别存放目标串p 和s中的子串的字母的出现次数**。

```python
class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        m,n = len(s),len(p)
        if m<n:
            return []
        left,right=0,0
        s_i=[0]*26
        p_i=[0]*26
        ans=[]

        for i in range(n):
            p_i[ord(p[i])-ord('a')]+=1 #先记录p的各个字母出现次数
        for right in range(m):
            s_i[ord(s[right])-ord('a')]+=1 #每枚举一个新的字母(s[right])，就更新一下次数
            if right>=n: #如果和p一样长了，就要移动left了
                s_i[ord(s[left])-ord('a')]-=1
                left+=1
            if p_i==s_i:# 只要字母的次数一样多
                ans.append(left)

        return ans
```

+ 注意ord方法的使用

## [62. 不同路径](https://leetcode.cn/problems/unique-paths/) 26/3/9
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1773027078123-1706a95b-38cb-4769-8b41-4d6a6c7d823d.png)

思路：

我们定义`dp[i][j]`  表示从start（假设为1,1）到第 i 行，第 j 列总共有多少条路径。

显然可以知道，转移方程为：`dp[i][j]=dp[i-1][j] + dp[i][j-1]`

对于dp问题，初始化很重要，由于出现了`i-1,j-1` 方便起见，我们规定**下标从1**开始（如果从0的话，0-1=-1，在python中相当于倒数第一个，不符合逻辑）

所以只需要初始化`dp[0][1]=1`即可，这样一来，经过转移方程，第一行全为1，第一列全为1，符合逻辑。

代码：

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        dp=[[0]*(n+1) for _ in range(m+1)]

        dp[0][1]=1
        # print(dp)
        for i in range(1,m+1):
            for j in range(1,n+1):
                dp[i][j]=dp[i-1][j]+dp[i][j-1]

        return dp[m][n]
```

再分析一下，能不能把dp数组压缩到一维呢？

当然可以！因为下一行的dp数组，只与上一行有关系！更准确的说，它只和正上方和左边的元素有关系。

所以我们只需要一维的dp数组即可，这样就能保留上一行的数据，从而在更新下一行dp的时候用到。

分析一下原来的转移方程：`dp[i][j]=dp[i-1][j]+dp[i][j-1]`

其中，`dp[i-1][j]` 就是上一行的数据，`dp[i][j-1]` 就是左边的数据。我们可以直接把 i 去掉

**转移方程为：**`**dp[j]=dp[j]+dp[j-1]**`

```python
for i in range(1,m+1):
    for j in range(1,n+1):
        dp[j]=dp[j]+dp[j-1]
```

+ 这里说一下为什么直接把 i 去掉并不影响。  
因为遍历是从左往右遍历的，而赋值是右边赋值左边的，当遍历到第i行 第j列的时候，此时右边的`dp[j]`还没更新，也就相当于是上一行留下来的（上方）;  
而`dp[j-1]`刚刚已经更新了，就相当于左边传过来的！

另外，对应的初始化也要修改，dp数组现在只有一维了，我们**不妨把第一行的先初始化为1**（本身第一行都是1，因为只能从左边过来），不过要注意，这么一来，**外层循环要少一次！因为第一行已经初始化了**！

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        dp=[0]+[1]*n

        # print(dp)
        for i in range(1,m):
            for j in range(1,n+1):
                dp[j]=dp[j]+dp[j-1] 
        return dp[n]
```

## [64. 最小路径和](https://leetcode.cn/problems/minimum-path-sum/) 26/3/10
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1773111653058-7b40b713-6cac-4712-9b94-8c7b87b92ee7.png)

思路：

这道题和不同路径那道题思路差不多的，都是只能向下向右移动，  
所以对应的`dp[i][j]`就只能由`dp[i-1][j], dp[i][j-1]`两个转移到

易知，**转移方程为：**`**dp[i][j]=min(dp[i-1][j],dp[i][j-1]) + grid[i-1][j-1]**`** 这里依然假设dp下标从1开始**。

那么应该如何**初始化**呢？分析可知，`dp[1][1]`应该等于`grid[0][0]`，那能不能直接`dp[1][1]=grid[0][0]`?不行。因为往后面算的话，会用到`dp[0][j], dp[i][0]` 也就是多出来的第0行第0列。那么怎么初始化能不受它们的影响呢？把它们初始化为无穷就好了！ 当然，也可以**单独把**`**dp[0][1]=0**`**，这样一来，**`**dp[1][1]**`**自然等于**`**grid[0][0]**`！

代码如下：

```python
class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        m,n=len(grid),len(grid[0])
        dp=[[float('inf')]*(n+1) for _ in range(m+1)]
        dp[0][1]=0
        for i in range(1,m+1):
            for j in range(1,n+1):
                dp[i][j] = min(dp[i-1][j],dp[i][j-1])+grid[i-1][j-1]
        # print(dp)
        return dp[m][n]
```

同样的，这题能不能压缩dp数组？

当然可以！

跟不同路径题目一样，转移方程为：`dp[j] = min(dp[j],dp[j-1])+grid[i-1][j-1]`

其中，在等式右边`dp[j-1]`代表刚更新过的新数据，也就是从左边传过来的；  
`dp[j]`表示上次更新的旧数据，也就是上一行的，从上面传过来的。

接下来该初始化了！

本来我有点想当然了，直接把dp等于grid[0]，然后dp[0]=inf；  
这显然是不对的，因为dp是最小路径之和！

所以从dp[1]开始，也就是对应第一行`grid[0][0]`开始往后，就该进入循环了（开始dp的递推）

那么我们**只需要保证第一次循环时，来自上层的数据，也就是等式右边的**`**dp[j]**`**不影响就行了，那不就是无穷大嘛**！ 

```python
class Solution:

        # print(dp)
```

## [5. 最长回文子串](https://leetcode.cn/problems/longest-palindromic-substring/) 26/3/10
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1773116462604-f87686f0-757f-4c5e-ad07-006113bedf64.png)

这道题是一个区间dp的问题。

区间dp的状态通常包含两个维度 `[i, j]`，表示在原序列中从下标 `i` 到下标 `j` 的这一段**区间**的状态。  

大区间 `[i, j]` 的状态，通常是由更小的、被它包含的子区间（比如 `[i+1, j-1]`、`[i, k]` 和 `[k+1, j]` 等）的状态合并或者推导而来的。 

区间 DP 不能像普通的线性 DP 那样简单地从左到右跑一个 `for` 循环。   你必须确保**“先计算短区间，再计算长区间”**  

那么对于这道题而言，我们定义`dp[i][j]`表示从 i 到 j 这个区间的子串是否为回文串。初始化自然要全部为False。在判断的时候，无非是下面两种情况：

+ **如果**`**s[i]!=s[j]**`**很显然**`**dp[i][j]=False**`
+ **当**`**s[i]==s[j]**`**，这个时候不能直接**`**dp[i][j]=True**`**，因为要考虑内部小区间的状态！**
    - **如果**`**i==j**`**，或者**`**j==i+1**`**，也就是**`**j-i<=1**`**这个时候肯定是回文串，因为最多只有两个字符！**
    - **反之，就要根据**`**dp[i+1][j-1]**`** 来判断！**注意这里回文串一定是中心对称的，如果外壳相等，就只考虑内部区间就好，不需要考虑`dp[i+1][j], dp[i][j-1]`这种

最后记得维护一个最大长度和左右端点。

代码如下

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        n=len(s)
        dp=[[False]*n for _ in range(n)]
        left,right=0,0
        maxlen=0

        for i in range(n-1,-1,-1):
            for j in range(i,n):
                if s[i]!=s[j]:
                    continue
                else:
                    if j-i<=1:
                        dp[i][j]=True
                    elif dp[i+1][j-1]:
                        dp[i][j]=True
                if dp[i][j] and j-i+1>maxlen:
                    maxlen=j-i+1
                    left=i
                    right=j
        return s[left:right+1]
```

## [1143. 最长公共子序列](https://leetcode.cn/problems/longest-common-subsequence/) 26/3/11
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1773197303193-f9f22773-1aac-4228-830c-a625c2bc40aa.png)

思路：

我们定义`dp[i][j]` 表示text1 前 i 个字符与text2前 j 个字符中最长的公共子序列长度。

当`text1[i]==text2[j]`的时候，说明找到了一个新的公共字符，最大长度 +1。问题是让谁 +1 ？  
答案是`dp[i-1][j-1]+1`。  
因为新的公共字符一定是在公共子序列的末尾的，所以让text1,text2都减去一个，才是它的上一个状态！

当`text1[i]!=text2[j]`，也就是这两个字符**不能同时**作为公共子序列的末尾。那么此时最大子序列的末尾自然要**分别从**`text1` 的前 i 个，`text2`的前 j 个中找最大的。`dp[i][j]=max(dp[i-1][j],dp[i][j-1])`

要求最大值，初始化为0即可。

代码如下：

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m,n=len(text1),len(text2)
        dp=[[0]*(n+1) for _ in range(m+1)]

        for i in range(1,m+1):
            for j in range(1,n+1):
                if text1[i-1]==text2[j-1]:
                    dp[i][j]=dp[i-1][j-1]+1
                else:
                    dp[i][j]=max(dp[i-1][j],dp[i][j-1])

        return dp[m][n]
        
```

这个问题能不能压缩dp数组？当然可以！

因为它也是只用到了上一行的数据！所以只用一维数组就可以表示 dp

再分析一下那两个转移方程：  
`dp[i][j]=max(dp[i-1][j],dp[i][j-1])`，这个很显然了，当一维dp遍历到`dp[j]`的时候（此时还没更新），右边的`dp[j-1]`就是刚刚更新的，相当于`dp[i][j-1]`; 右边的`dp[j]`是还没更新的相当于上一行的`dp[i-1][j]`。所以，转移方程等价于 `dp[j]=max(dp[j],dp[j-1])`！！！

对于 `dp[i][j]=dp[i-1][j-1]+1` 就稍微有点棘手，因为要用到左上角的元素，而在一维数组中，遍历到`dp[j]`时，`dp[j-1]`一定会被更新，也就是只有一个数组是没办法保存上一行（左上角）的元素的！那么就可以再增加两个变量`pre` `tmp`，`pre`在每个外层循环赋值为0，代表着左上角初始为0（这是肯定的，因为我们dp数组从1开始）。而`tmp`记录每次更新`dp[j]`之前它的值，相当于`dp[i-1][j]`。重点来了！** 只要在每个内层循环 j 结束之后，让**`**pre=tmp**`**，进入下一轮的j循环后，pre就相当于**`**dp[i-1][j-1]**`**！！！**

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m,n=len(text1),len(text2)
        dp=[0]*(n+1)

        for i in range(1,m+1):
            pre=0 #存左上角的值

            for j in range(1,n+1):
                tmp=dp[j] #存当前正在遍历的值，但是还没覆盖，所以代表dp[i-1][j]；在下一轮就变成了左上角的值
                if text1[i-1]==text2[j-1]:
                    dp[j]=pre+1
                else:
                    dp[j]=max(dp[j],dp[j-1])
                pre= tmp

        # print(dp)
        return dp[n]
```

## [72. 编辑距离](https://leetcode.cn/problems/edit-distance/) 26/3/11
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1773200989389-3534e176-dfb9-4783-9def-dec16e34a0f1.png)

这道题是有点抽象的，刚开始也没想到怎么能用dp表示出来，后面看了一下题解就理解了。

我们先按照直觉，定义`**dp[i][j]**`**表示源字符串**`**word1**`**前**$ i $**个字符转换到目标字符串**`**word2**`**前**$ j $**个字符所需要的最少操作数。**

下面我们分析一下转移方程：

第一种情况，当`word1[i]==word2[j]`，这个时候，`word1`前$ i $个字符转换到`word2`前$ j $个字符所需要的操作数，其实和`word1`前$ i-1 $个字符转换到`word2`前$ j-1 $个字符是一样的！也就是不需要增加新的操作。即`dp[i][j]=dp[i-1][j-1]`

当`word1[i]!=word2[j]`，有下面三种操作：

+ 替换

也就是先将`word1`前$ i-1 $个字符转换为`word2`前$ j-1 $个字符。在此基础上，只需要对`word1`的第$ i $个字符执行一次替换就好！  
`dp[i][j]=dp[i-1][j-1]+1`

+ 插入

先将`word1`前$ i $个字符转换为`word2`前$ j-1 $个字符。在此基础上，想得到`dp[i][j]`，也就是让`word1`前$ i $个字符转换为`word2`前$ j $个字符，只需要在`word1`后面插入一个新的元素即可。  
`dp[i][j]=dp[i][j-1]+1`

+ 删除

先将`word1`前$ i-1 $个字符转换为`word2`前$ j $个字符。想得到`dp[i][j]`，也就是让`word1`前$ i $个字符转换为`word2`前$ j $个字符，只需要在`word1`后面删除一个新的元素即可（因为前 i-1 已经转换成功了）

但是以上三种操作不是同时进行的，肯定要取最小值！

另外，初始化的时候，`dp[i][0]=i`，因为`word1`前$ i $个字符转换为`word2`前$ 0 $个字符，肯定要操作i次（都删了）；同理`dp[0][j]=j`，（都插入）

代码如下：

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m,n=len(word1),len(word2)
        dp=[[0]*(n+1) for _ in range(m+1)]
        for i in range(m+1):
            dp[i][0]=i
        for j in range(n+1):
            dp[0][j]=j

        for i in range(1,m+1):
            for j in range(1,n+1):
                if word1[i-1] == word2[j-1]:
                    dp[i][j]=dp[i-1][j-1]
                else:
                    dp[i][j]=min(dp[i-1][j-1],dp[i-1][j],dp[i][j-1])+1
        return dp[m][n]
```
