# 预训练
## lm_dataset
先说一下Dataset和DataLoader

**Dataset，实现了按需加载数据**

它是 PyTorch 中的一个类 (`torch.utils.data.Dataset`)。定义的任何自定义数据集（比如`PretrainDataset`）都必须继承它且 必须实现两个核心方法：

+ `**__len__(self)**`：告诉调用者这个数据集里**总共有多少条数据**。  
+ `**__getitem__(self, index)**`：定义了当给定一个索引 `index` 时，**如何读取并处理这“一条”数据**。  

**DataLoader（ 数据加载器  ）,实现把数据批次加载进去**

它是 PyTorch 中早已写好的一个工具类 (`torch.utils.data.DataLoader`)，只需把实例化好的 `Dataset` 传给它，并设置几个关键参数  

+ `batch_size`（批次大小）  
+ `shuffle`（打乱顺序）通常在训练集上设为 `True`，防止模型死记硬背数据的顺序；在测试集中设为 `False`
+ `num_workers`（工作进程数）  
+ `collate_fn`（打包函数）  



**预训练数据处理流程如下：**

+ **先提取出text**
+ **然后添加bos,eos**
+ **转换为tokenid**
+ **得到label**

```python
class PretrainDataset(Dataset):
    def __init__(self, data_path, tokenizer, max_length=512):
        super().__init__()
        self.tokenizer = tokenizer #分词器
        self.max_length = max_length 
        self.samples = load_dataset('json', data_files=data_path, split='train')#加载数据集到self.samples，data_path是json文件的路径，split='train'表示加载训练集

    def __len__(self):
        return len(self.samples)

    def __getitem__(self, index):
        sample = self.samples[index] #根据索引获取数据

        # 提取到文本后，进行分词，转为tokenid
        # add_special_tokens=False表示不添加特殊token(我们自己添加)
        # max_length=self.max_length - 2表示文本的最大长度，减去2是因为我们要添加bos_token和eos_token
        # .input_ids表示只获取分词后的tokenid列表,因为我们要手搓
        tokens = self.tokenizer(str(sample['text']), add_special_tokens=False, max_length=self.max_length - 2, truncation=True).input_ids
        # 在分词列表的头部加上 bos_token_id，尾部加上 eos_token_id
        tokens = [self.tokenizer.bos_token_id] + tokens + [self.tokenizer.eos_token_id]
        # 补齐到max_length，使用pad_token_id进行填充
        input_ids = tokens + [self.tokenizer.pad_token_id] * (self.max_length - len(tokens))
        # 把list转为torch.tensor，并且设置数据类型为long
        input_ids = torch.tensor(input_ids, dtype=torch.long)

        # 因为自回归模型的输入数据就是标签
        # 我们直接复制一份，模型内部会进行shift对齐
        labels = input_ids.clone()
        # 贴标签：将padding部分的标签设置为-100，模型在计算损失时会忽略这些位置
        labels[input_ids == self.tokenizer.pad_token_id] = -100
        return input_ids, labels
```

## train_pretrain
**先来说一下epoch，step，batchsize的关系**

+ batch size：批次大小，也就是dataloader一次加载过来的多少批次
+ step/iteration：迭代步数，**是指模型进行一次前向传播、反向传播并且更新一次权重的过程**。没训练完batch，就走一个step。
+ epoch：把整个数据集训练了一遍，就是一个epoch

**一个epoch的step数量 = 数据集总大小 / batch size = 数据集有几个batch**

**训练的核心流程：**

+ **前向传播**
+ **计算loss**
+ **反向传播：计算出参数的梯度**
+ **梯度下降：用梯度去更新权重，使得loss变小（理想情况下）**

**梯度累积：**

当我们的batchsize太小，而显存又不够大的时候。我们可以叠加多个batch的loss，然后放一块去进行梯度下降！就相当于用来大的batchsize！

为什么要把batchsize变大？

+ 注意` loss = loss / args.accumulation_steps` 因为 PyTorch 算出的 Loss 默认是一个 Batch 的平均值，必须先除以要累积的倍数，后面就可以直接加起来了。
+ 用`if (step + 1) % args.accumulation_steps == 0:`判断，就能实现梯度累积

**混合精度：**

简单来说就是，32位的数据质量高，但是占用内存多；而16位质量差占用内存少。所以让一部分不太“敏感”的数据暂时用16位存，在不影响质量的情况下，减少的内存！

+ 16位： 矩阵乘法（`Linear`、`matmul`、`bmm`）、卷积（`Conv1d/2d/3d`）  
+ 32位：`Softmax`、`CrossEntropyLoss`...

放在训练过程中，在前向传播、计算loss、反向传播的时候，都会有16位的数据存在。当`loss.backward()`结束后，所有数据变为32位。然后进行梯度下降，更新权重。

所以在反向传播的时候，要把loss扩大，避免出现梯度下溢！

之后进行梯度下降的时候，再把优化器给还原回来

核心代码如下：

```python
def train_epoch(epoch, loader, iters, start_step=0, wandb=None):#start_step 表示断点恢复时从哪个 step 后继续
    start_time = time.time() #记录本轮 epoch 的开始时间

    # 遍历数据加载器，获取每个 batch 的输入和标签，step 从 start_step + 1 开始计数，确保断点恢复时不会重复训练已经完成的步骤
    for step, (input_ids, labels) in enumerate(loader, start=start_step + 1):
        input_ids = input_ids.to(args.device) #把输入数据和标签都放到gpu
        labels = labels.to(args.device)
        lr = get_lr(epoch * iters + step, args.epochs * iters, args.learning_rate) # 余弦退火学习率
        for param_group in optimizer.param_groups:
            param_group['lr'] = lr #更新优化器的学习率

        ''' 前向传播 + loss'''
        with autocast_ctx: #这是一个上下文管理器，目的是让前向传播计算loss的时候使用bf16
            res = model(input_ids, labels=labels)
            # loss = res.loss + res.aux_loss
            loss = res.loss
            loss = loss / args.accumulation_steps #梯度累积？？？

        ''' 反向传播：计算完loss之后，要把低精度的loss放大,避免反向传播梯度下溢（这里也是16位）。'''
        scaler.scale(loss).backward() # 在backward结束后，变为32位

        ''' 梯度下降 '''
        if (step + 1) % args.accumulation_steps == 0:
            scaler.unscale_(optimizer) #先还原梯度
            torch.nn.utils.clip_grad_norm_(model.parameters(), args.grad_clip)

            scaler.step(optimizer) #更新权重，注意优化器的参数是32位的
            scaler.update()

            optimizer.zero_grad(set_to_none=True) #清空梯度，set_to_none=True可以节省内存
```

<|im_start|>assistant

+ 数据清洗：把无意义的空推理标签去掉
+ 转换为tokenid
+ 构造label

+ 初始权重, `--from_weight`改为`pretrain`

**另外，关于学习率，这里总结一下：**

在**预训练的时候，模型越大，学习率就要越小**。如小模型（<1B，学习率可以用`1e-4`~`6e-4`；7B左右，可以用`1e-5`~`3e-4`；70B+，可以用`6e-5`甚至更低。

SFT的学习率一般**要比预训练小一到两个数量级，一般**`**1e-5**`**~**`**1e-6**`

**PEFT/LoRA学习率要比SFT大，一般**`**1e-4**`**~**`**5e-4**`

## PEFT 之 LoRA
Low Rank，低秩。

秩是真正能把矩阵支撑起来的维度。

在微调中，不需要满秩，**因为他们具有非常低的内在秩**。

由奇异值分解的知识可知，任何一个矩阵，都可以分解成三个矩阵相乘。

$ A_{m \times n} = U_{m \times m} \Sigma_{m \times n} V^T_{n \times n} $

+ $ U $** (左奇异向量)：** 一个正交矩阵，代表了输出空间的基向量（可以理解为旋转操作）。
+ $ \Sigma $** (奇异值矩阵)：** 一个对角矩阵。对角线上的元素叫“奇异值”，**按从大到小的顺序排列**。奇异值的大小直接反映了该维度特征的重要程度（可以理解为拉伸/缩放操作）。
+ $ V^T $** (右奇异向量)：** 另一个正交矩阵，代表了输入空间的基向量（也是旋转操作）。

在真实世界的数据（无论是图像还是大模型的权重）往往存在大量冗余。在 $ \Sigma $ 矩阵中，通常只有前几个奇异值非常大，后面的奇异值接近于零。故低秩近似后有：

$ A_{m \times n} \approx U^{new}_{m \times r} \Sigma^{new}_{r \times r} V^T_{r \times n} $，

合并后面三个矩阵到两个：$ A_{m \times n} \approx [\text{Matrix}]_{m \times r} [\text{Matrix}]_{r \times n} $

由此，我们可以想到，对于大模型的参数矩阵$ W $可以直接用两个小矩阵$ A $, $ B $来替代！

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/51029207/1773321330163-94d59163-5f3f-4da9-a0d4-0e52bf0b75a9.png)

LoRA的思路便来源于此，那么它具体原理是什么呢？

如上图所示：lora设计了两个矩阵矩阵 $ A $（下方的矩阵 ($ d \times r $)）,矩阵 $ B $（上方的矩阵 ($ r \times d $)）

对于一个输入的batch，lora借鉴了残差连接的思想，一边走预训练的分支，一边走lora的分支，通过更新$ \Delta W = AB $，来确定微调后的参数矩阵$ W_{new} = W_0 + A B $。

注意在初始化时，矩阵 $ A $ 用随机高斯分布初始化，只有矩阵 $ B $ 初始化为 0。

为什么B为0？因为**在微调的最初始阶段，模型和未微调的预训练模型表现完全一模一样**。这是一种非常安全的策略，防止让模型遗忘知识。

而且这样并不会影响梯度的更新，因为A不为0，**对矩阵 **$ B $** 的梯度**：$ \frac{\partial L}{\partial B} = (xA)^T \frac{\partial L}{\partial y} $，就不为0，所以B在第二轮就更新了！

另外，lora在模型推理的时候还有一个巨大的优势。因为我们在内存中可以直接把训练好的$ AB $矩阵和

$ W_0 $合并！所以模型推理没有额外的负担！





现在lora很简单，用peft很方便。

下面只需要明确两个问题：lora注入到哪里；rank秩等于多少

实验表明，将lora 注入 Wq，Wv 效果最好。不过GLM中Q,K,V都有注入
